🔍 שכבה 2 — שיפור שליפה

Retrieval — מה Dense Embedding לא יכול לעשות

Dense embeddings מעולים לסמנטיקה — אבל גרועים לחיפוש מספרים מדויקים, שמות, וביטויים ספציפיים. כאן נכנסת שיטות משלימות.

4. Hybrid Retrieval — BM25 + Dense

📅 יום

BM25 הוא מנוע חיפוש lexical — הוא מצטיין בדיוק על מספרים וביטויים מדויקים. שילוב עם Dense via RRF (Reciprocal Rank Fusion) מכסה את שתי החולשות.

✓ ישיר לכשלי Q1/Q4 ✓ ROI גבוה ביותר ⚠ תוספת תלות: rank-bm25
❌ Dense בלבד
שאילתה: "פרמיה פנסים" → מוצא chunks על ביטוח כולל, לא שורה ספציפית
✅ BM25 + Dense
BM25 מוצא 58 ו-פנסים ומראות בדיוק; Dense מוסיף הקשר סמנטי 
from rank_bm25 import BM25Okapi

# RRF — Reciprocal Rank Fusion
def rrf_merge(dense_ids, bm25_ids, k=60):
    scores = {}
    for rank, doc_id in enumerate(dense_ids):
        scores[doc_id] = scores.get(doc_id, 0) + 1/(k + rank + 1)
    for rank, doc_id in enumerate(bm25_ids):
        scores[doc_id] = scores.get(doc_id, 0) + 1/(k + rank + 1)
    return sorted(scores, key=lambda x: scores[x], reverse=True)

Chunking — הפתרון המלא לשורות טבלה

הפתרון האמיתי ל-Q1/Q2/Q4. שורות טבלה הן יחידות מידע עצמאיות — הן צריכות להיות chunks עצמאיים.

7. Table-Aware Chunking — שורה = chunk

🗓 שבוע

כרגע: טבלה שלמה (עשרות שורות) = chunk אחד גדול. המודל לא יכול "לאחזר" שורה ספציפית. הפתרון: זיהוי טבלאות Markdown, ויצירת chunk לכל שורה עם header context חוזר.

✓ פותר Q1 + Q4 ישירות ✓ ייתכן גם Q2 ✗ מורכבות גבוהה ב-parser
❌ היום — chunk אחד לכל טבלה
## פרמיות
| סוג כיסוי | עלות שנתית |
| גוף הרכב | 1,200 ש"ח |
| פנסים ומראות | 58 ש"ח | ← אבוד בתוך 2,800 תווים
| השתתפות עצמית | 2,000 ש"ח |
✅ מה רצוי — chunk לכל שורה
chunk: "פרמיות / פנסים ומראות צד: 58 ש"ח"

chunk: "פרמיות / גוף הרכב: 1,200 ש"ח"

chunk: "פרמיות / השתתפות עצמית: 2,000 ש"ח" 
def chunk_table_rows(markdown_table: str, section: str, doc: str):
    lines = markdown_table.strip().splitlines()
    headers = [h.strip() for h in lines[0].split("|") if h.strip()]
    for row_line in lines[2:]:  # skip header + separator
        cells = [c.strip() for c in row_line.split("|") if c.strip()]
        row_text = ", ".join(
            f"{h}: {v}" for h, v in zip(headers, cells)
        )
        yield make_chunk(f"{section} / {row_text}", doc)
